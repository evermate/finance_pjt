## 커밋 메시지 컨벤션

커밋 메시지는 아래와 같은 형식을 따릅니다.

### ✅ 타입 종류

| 타입       | 설명                     |
|------------|--------------------------|
| `feat`     | 새로운 기능 추가         |
| `fix`      | 버그 수정                |
| `docs`     | 문서 수정 (README 등)    |
| `refactor` | 코드 리팩토링 (기능 변화 없음) |
| `style`    | 코드 스타일 수정 (공백, 세미콜론 등) |
| `test`     | 테스트 코드 추가 및 수정 |

### 🧩 예시

feat: 로그인 기능 추가  
fix: 회원가입 오류 수정  
docs: README 업데이트  
refactor: 중복 코드 리팩토링  
style: 들여쓰기 수정  
test: 로그인 테스트 코드 추가
